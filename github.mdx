### Git/GitHub Repositories
A <b style={{color:`slateblue`}}>repository</b> is like a Folder on your computer.<br/><br/>
A repository holds your Project and changes.<br/><br/>
The only difference between a usual folder and a Git repository is that it contains a (*hidden*) `.git` folder.

---
A GitHub Repository can hold anything from Code to Multimedia Files to large binaries. <br></br>The upper limit on the size of a repo being 1 GB

### GitHub Pull Requests
Pull Request are a way through which developers collaborate on GitHub.<br/><br/>They provide a user-friendly web interface for discussing proposed changes before integrating them into the official project.

--- 

Pull requests lets you tell others about changes you've pushed to a GitHub repository. 

Once a pull request is sent, interested parties can review the set of changes, discuss potential modifications, and even push follow-up commits if necessary.

---
Pull Requests basically work with <code style={{color:`red`}}>branchesá›˜</code> and <code style={{color:`yellow`}}>forksâ‘‚.</code><br/>

Either you can open a PR from another branch or from a fork.
